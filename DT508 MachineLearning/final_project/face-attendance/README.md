# Face Attendance

ระบบเช็คชื่อเข้า–ออกห้องเรียนด้วยการจดจำใบหน้า (Face Recognition)  
สร้างด้วย **Flask** + **DeepFace** + **SQLite**

---

## คุณสมบัติ

- เปิดกล้องจากเบราว์เซอร์ แล้วสแกนใบหน้าเพื่อบันทึก **เข้า (IN)** / **ออก (OUT)**
- เพิ่มนักเรียนพร้อมอัปโหลดรูปใบหน้า
- แสดงรายชื่อนักเรียน ประวัติการเข้า–ออก และรายชื่อที่เข้าวันนี้
- เก็บเวลาเป็นโซน **Asia/Bangkok (UTC+7)**
- สร้าง face embedding ตอนเพิ่มนักเรียน เพื่อให้สแกนครั้งถัดไปเร็วขึ้น
- กันกดสแกนซ้ำขณะระบบกำลังประมวลผล

---

## สแต็กที่ใช้

| ส่วน | เทคโนโลยี |
|------|-----------|
| Backend | Flask |
| Face AI | DeepFace (โมเดล `VGG-Face`, detector `retinaface`) |
| Database | SQLite (`database/attendance.db`) |
| Frontend | HTML + Bootstrap 5 + JavaScript |
| Runtime | Python ≥ 3.11 (แนะนำจัดการแพ็กเกจด้วย [uv](https://github.com/astral-sh/uv)) |

```
face-attendance/

├── templates/
│   └── index.html       # หน้า UI หลัก
├── static/
│   └── script.js        # กล้อง, สแกน, โหลดรายการ, กันกดซ้ำ
├── faces/               # รูปใบหน้านักเรียน + ไฟล์ .pkl ของ DeepFace
├── database/
│   └── attendance.db    # ฐานข้อมูล SQLite
├── models/              # (ถ้ามี) น้ำหนักโมเดลที่ดาวน์โหลดเพิ่ม
```

### บทบาทไฟล์หลัก

| ไฟล์ | หน้าที่ |
|------|---------|
| `main.py` | เสิร์ฟหน้าเว็บ, API, เสิร์ฟรูปจาก `/faces/`, เรียก `face_utils` + `database` |
| `database.py` | สร้างตาราง, CRUD นักเรียน, บันทึก attendance, query ประวัติ / วันนี้ |
| `face_utils.py` | sanitize ชื่อไฟล์ (hex), detect / recognize, refresh embeddings |
| `static/script.js` | เปิดกล้อง, ส่งรูปไป API, แสดงสถานะและรายการ |

### API หลัก

| Method | Path | คำอธิบาย |
|--------|------|----------|
| `GET` | `/` | หน้าเว็บหลัก |
| `GET` | `/faces/<filename>` | เสิร์ฟรูปใบหน้า |
| `GET` | `/api/students` | รายชื่อนักเรียนทั้งหมด |
| `POST` | `/api/students/add` | เพิ่มนักเรียน (form: `name`, `image`) |
| `POST` | `/api/recognize` | สแกนใบหน้า (form: `image`, `action=IN\|OUT`) |
| `GET` | `/api/attendance/history` | ประวัติเข้า–ออก (`?limit=20`) |
| `GET` | `/api/attendance/today` | คนที่เข้าวันนี้ |

## รันบนเครื่องอื่น (มีเน็ต)

ไม่จำเป็นต้องเอาไฟล์โมเดลขนาดใหญ่ (เช่น `models/deepface/*.h5`) ไปด้วย  
DeepFace จะ **ดาวน์โหลดน้ำหนักโมเดลอัตโนมัติ** เมื่อใช้งานใบหน้าครั้งแรก

### ขั้นตอน

```powershell
git clone https://github.com/fhunsiwakorn/face-attendance.git
cd face-attendance
uv sync
uv run python main.py
```

เปิดเบราว์เซอร์ที่ http://127.0.0.1:5000

### ตอนไหนที่ดาวน์โหลดโมเดล

การรัน `main.py` อย่างเดียวอาจยังไม่ดาวน์โหลดครบ  
โมเดลจะถูกดาวน์โหลดเมื่อ:

1. **เพิ่มนักเรียน** (ตรวจใบหน้า + สร้าง embedding) หรือ
2. **สแกนเข้า/ออก** ครั้งแรก

ครั้งแรกจะช้ากว่าปกติ (ดาวน์โหลดไฟล์ใหญ่ ~500MB+)  
ครั้งถัดไปจะใช้ไฟล์ที่โหลดไว้แล้ว ไม่ต้องดาวน์โหลดใหม่

ไฟล์มักถูกเก็บที่:

- `models/deepface/` ในโปรเจกต์ หรือ
- `C:\Users\<ชื่อผู้ใช้>\.deepface\weights\` (Windows)

### หมายเหตุ

- เครื่องต้องมีเน็ตเสถียรตอนใช้งานใบหน้าครั้งแรก
- ไม่ควร commit ไฟล์ `.h5` ขึ้น GitHub (เกิน 100MB) — ให้ ignore แล้วให้แต่ละเครื่องดาวน์โหลดเอง
- เครื่องที่ไม่มีเน็ตต้องคัดลอกโฟลเดอร์น้ำหนักโมเดลไปด้วยเอง

## วิธีใช้งาน

1. เปิดหน้าเว็บ → กด **เปิดกล้อง** และอนุญาตสิทธิ์กล้อง
2. แท็บเพิ่มนักเรียน → กรอกชื่อ + เลือกรูปใบหน้าชัดๆ → บันทึก  
   (รอจนระบบสร้าง embedding เสร็จ)
3. กด **เข้า** หรือ **ออก** เพื่อสแกนใบหน้า  
   - อย่ากดซ้ำขณะขึ้นข้อความ “กำลังสแกน…”
4. ดูผลได้ที่รายชื่อนักเรียน / ประวัติ / คนเข้าวันนี้

### เคล็ดลับให้จดจำแม่นขึ้น

- ใช้รูปหน้าตรง แสงพอ ใบหน้าชัด
- ตอนสแกนหันหน้าเข้ากล้องในมุมใกล้เคียงรูปที่ลงทะเบียน
- หลังแก้โค้ดหรืออัปเดต dependencies แนะนำ hard refresh หน้าเว็บ (`Ctrl + F5`)

## การออกแบบสำคัญในโค้ด

### เวลา (Timezone)

SQLite `CURRENT_TIMESTAMP` เป็น UTC ทำให้เวลาไทยเพี้ยน (เช่น แสดง 17:10 แทน 00:10)  
โปรเจกต์จึงบันทึกเวลาด้วย `Asia/Bangkok` ใน `database.py` (`now_bangkok()` / `today_bangkok()`)

### ชื่อไฟล์รูป

ชื่อนักเรียนภาษาไทยถูกแปลงเป็น hex ก่อนบันทึกใน `faces/`  
เพื่อเลี่ยงปัญหา non-ASCII path ของ DeepFace แล้ว map กลับชื่อจริงผ่านฐานข้อมูล

### Face recognition

- โมเดล: `VGG-Face`
- Detector: `retinaface` (เร็วและเสถียรกว่า `mtcnn` ในเครื่องส่วนใหญ่)
- มี lock กันสแกนซ้อน และสร้าง embedding ตอนเพิ่มนักเรียนผ่าน `refresh_embeddings()`

### การเสิร์ฟรูป

Flask มี route `/faces/<filename>` เพื่อให้ UI แสดงรูปนักเรียนได้ถูกต้อง

## แนวทางต่อยอด

ไอเดียที่ต่อได้โดยไม่ต้องเปลี่ยนสถาปัตยกรรมมาก:

1. **ลบ / แก้ไขนักเรียน** — API + ปุ่มใน UI พร้อมลบไฟล์ใน `faces/` และ refresh embeddings
2. **หลายใบหน้าต่อคน** — เก็บหลายรูปต่อ `student_id` เพื่อให้จดจำได้ในหลายมุม
3. **เกณฑ์ความมั่นใจ (threshold)** — ปฏิเสธผลที่ confidence ต่ำเกินไป ลดการจับคู่ผิด
4. **ส่งออกรายงาน** — CSV / Excel ของประวัติรายวัน รายวิชา
5. **Login / บทบาท** — แยกสิทธิ์ครู–แอดมิน ก่อนจัดการรายชื่อ
6. **โมเดลเบาลง** — ทดลอง `Facenet512` หรือ detector อื่นเพื่อความเร็วบนเครื่องสเปกต่ำ
7. **Deploy จริง** — ใช้ Waitress / Gunicorn แทน Flask debug server และปิด debug
8. **กล้องหลายจุด** — ระบุ `device_id` หรือตำแหน่งห้องตอนบันทึก attendance

## การแก้ปัญหาเบื้องต้น (Troubleshooting)

| อาการ | แนวทางตรวจ |
|-------|-------------|
| เปิดกล้องไม่ได้ | ตรวจสิทธิ์กล้องของเบราว์เซอร์ / ใช้ HTTPS หรือ `localhost` |
| เพิ่มนักเรียนแล้วขึ้น “ไม่พบใบหน้า” | ใช้รูปหน้าชัดขึ้น หรือเปลี่ยนแสง |
| สแกนแล้วไม่ตรงคน | ลงทะเบียนใหม่ด้วยรูปที่คล้ายมุมกล้องจริง |
| รูปในรายชื่อไม่ขึ้น | ตรวจว่าไฟล์ยังอยู่ใน `faces/` และ path ใน DB เป็น `faces/...jpg` |
| ครั้งแรกช้ามาก | ปกติ — DeepFace กำลังดาวน์โหลด/โหลดโมเดล |
| Error เรื่อง OpenCV Haar | โปรเจกต์ใช้ `retinaface` อยู่แล้ว ไม่ควรพึ่ง `opencv` detector บน OpenCV 5 |

ถ้า embeddings เพี้ยน สามารถลบไฟล์ `faces/*.pkl` แล้วเพิ่มนักเรียนใหม่ หรือเรียก `refresh_embeddings()` อีกครั้งได้
